# 只读调查规则

这些规则对该技能的每一次使用都具有强制性。

## 允许

- 读取日志。
- 使用 `rg`、`grep` 与 `find` 搜索代码。
- 查看路由（Route）、控制器（Controller）、视图（View）、服务（Service）、模型（Model）、仓储（Repository）、中间件（Middleware）、序列化器（Serializer）、模式（Schema）、配置读取器与客户端代码。
- 分析调用链路。
- 使用只打印结果的只读 shell 管道。
- 汇总证据并对可能原因进行排序。

## 禁止

- 修改代码。
- 修改配置。
- 创建、删除、移动、复制或重命名文件。
- 运行 `rm`、`mv` 或 `cp`。
- 运行 `sed -i`。
- 执行数据库写操作。
- 运行 `migrate` 或会改变 schema 的命令。
- 执行清缓存、重启队列、重启服务、部署，或任何会改变系统状态的命令。
- 输出修复代码或实现补丁。

## 命令安全策略

允许的命令模式包括：
- `rg ...`
- `grep ...`
- `find ...`
- `ls ...`
- `cat ...`
- `sed -n ...`
- `head ...`
- `tail ...`
- `awk ...`
- `sort ...`
- `uniq ...`

不要运行包含原地编辑、破坏性参数，或向项目文件写入重定向的命令。

## 范围规则

在可能的情况下，搜索时始终排除以下目录：
- `vendor`
- `node_modules`
- `.git`
- `__pycache__`
- `venv`

优先按以下顺序使用仓库内证据：
1. 日志
2. 路由定义
3. 入口处理器（Handler）
4. 服务（Service）代码
5. 模型（Model）或仓储（Repository）代码
6. 配置读取器与外部客户端封装

## 报告规则

- 只做诊断。
- 不要提出代码修复方案。
- 每个可能原因都必须指向具体文件，以及符号或行上下文。
- 当证据不完整时，要明确说明不确定性。
- 验证建议保持只读。
